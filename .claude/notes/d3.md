# D3 Visualisation Learnings

### [2026-04-09] D3 canvases must use ResizeObserver, not fixed dimensions

**Context**: All four D3 components (ArcExplorer, TopicGalaxy, SessionTimeline, ArgumentMap) had hard-coded width/height defaults (800x600, etc). Pages didn't pass explicit sizes.
**Correction**: Created `useContainerSize` hook using ResizeObserver with a callback ref pattern. Each component wraps its SVG in a measured div. Pages use flex layout (`flex-1 min-h-0`) to fill available space. Layout.tsx changed from `min-h-screen` to `h-screen` with `overflow-hidden`.
**Rule**: D3 visualisation components must never have hard-coded dimensions. Use ResizeObserver to measure the container and pass measured values to the SVG and simulation.
**Applies to**: apps/web/components/graph/*, apps/web/lib/hooks/use-container-size.ts

### [2026-04-09] React 18 ref types — use callback refs for ResizeObserver

**Context**: Initial `useContainerSize` hook returned `RefObject<T | null>` which caused TS2322 errors because React 18's `LegacyRef` expects `RefObject<T>` (no null in generic).
**Correction**: Changed to a callback ref pattern `(node: T | null) => void` which avoids the type mismatch entirely and also handles element mounting/unmounting cleanly.
**Rule**: When creating custom hooks that observe DOM elements, prefer callback refs over `useRef` to avoid React 18 ref type compatibility issues.
**Applies to**: apps/web/lib/hooks/use-container-size.ts
